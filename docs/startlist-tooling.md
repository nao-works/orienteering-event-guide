# スタートリスト生成ツールの役割

この repo には Oスタと Oガイドの公開ブラウザ版を置く。主な開発対象は Oスタ。Oガイドは公開済みデータの見え方を試す実験的な情報ビューであり、継続更新される大会情報サービスとしては扱わない。

スタートリスト生成には、ブラウザで即時に使える便利さと、大会運営の正本を作るための再現性の両方が必要になる。この二つは同じものではない。

## Oスタ

Oスタは、JOY / Japan-O-entrY 形式の CSV をブラウザで読み込み、締切後にスタートリストを作るための単日大会向け簡易ツール。

向いていること:

- 入力 CSV の確認
- 生成設定とスタート時刻の整理
- 公開前チェック
- 単日大会のスタートリスト下書き
- クラス別人数などの集計メモ作成
- クラス順やレーンの試行
- WRE / ランキング順スタートの試行
- Mulka CSV 形式のたたき台作成

現時点で弱いこと:

- イベント固有ルールの明示的な設定管理
- 入力ファイル、seed、生成条件、警告、出力件数をまとめた manifest
- 重複カード番号、空カード、固定ゼッケン、フリースタート、複数日大会の体系的な検証
- 補正履歴の追跡

したがって、Oスタ単体の出力を大会運営の最終正本として扱わない。

## Oガイド

Oガイドは、公開データの見え方と参加者向けUIを試すための情報ビュー。

向いていること:

- 公開済みデータの表示確認
- 参加者向けの導線や見せ方の試行
- Oスタや運営向けフローから出る公開情報の受け皿検討

現時点で弱いこと:

- 継続的な大会情報更新
- 情報の網羅性
- 主催者公式情報との常時同期

したがって、Oガイドを大会情報サービスとして押し出さない。参加・運営判断には主催者の公式情報を確認する。

## 運営向け正本生成フロー

大会運営の正本生成は、入力検査、イベント設定、補正履歴、生成記録を残せる運営向けフローを基準にする。

必要な性質:

- JOY 入力の正規化
- YAML などによるイベント固有ルール管理
- corrections / start preferences の記録
- 複数日、複数レーン、フリースタート、固定ゼッケンへの対応
- validate による事前検査
- Mulka CSV、公開 CSV / Markdown の生成
- 生成記録による再現性の確保

現在の内部実装では、このフローを CLI / core workflow として管理している。ただし、オリエンテーリング大会の主催者や運営者は非エンジニアであることが多いため、公開利用者に CLI の理解を求める形にはしない。

この workflow は、参加者データや大会固有の補正を扱うため、公開する場合はサンプルデータと生成済み出力を匿名化してから切り出す。

公開用の匿名サンプルは `core-sample/` に置いている。これは、実名を含まない JOY 風入力、イベント設定、日別スタートリスト、Mulka CSV、manifest を一式で見せるためのもの。

## 方針

1. Oスタはブラウザ版の簡易・下書きツールとして維持する。
2. Oガイドは実験的な情報ビューとして期待値を下げる。
3. 正本生成の仕様は運営向け正本生成フローを基準にする。
4. Oスタを強化する場合は、内部 core と同じ検証観点や fixture を共有する。
5. 内部 CLI を公開する場合は、実名入りサンプルや生成済み出力を含めない。
