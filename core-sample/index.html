OスタCoreは、ブラウザだけで試すOスタLiteとは別の考え方です。 JOY入力、イベント設定、補正履歴、日別スタートリスト、Mulka CSV、manifestをまとめて扱い、 大会運営で後から説明できる状態を作ります。

Lite と Core の違い

OスタLite

CSV確認と簡易下書きに向いたブラウザツールです。単日や小規模な試行には早く使えます。

OスタCore

2日間、レーン、フリースタート、固定ゼッケン、補正履歴、警告、生成記録を残す正本生成の考え方です。

生成の流れ

01
JOY風TSVを読む申込行と参加者を分けて把握します。
02
イベント設定を適用する日別、レーン、クラス順、フリースタート、ゼッケン開始番号を設定します。
03
補正と希望を記録するクラス変更や早め/遅め希望を設定ファイルに残します。
04
出力と警告を残す公開リスト、Mulka CSV、manifestを生成し、取込前の警告を確認します。

このサンプルで確認できること

日別出力Day1 10スタート、Day2 10スタートに分けています。
複数レーンLane 1 と Lane 2 で別々に時刻を進めます。
固定ゼッケン2日間出る人やチームは同じ番号を維持します。
補正履歴グループ内の1名だけをDay2に別クラスへ移す例を含みます。
Mulka取込前警告レンタルカード未割当をDay1 3件、Day2 5件としてmanifestに残します。
正本と下書きの分離これはOスタLiteを巨大化するためではなく、Coreの検証観点を見せるためのサンプルです。

コマンド形

現時点では実行CLI本体はこの公開リポジトリに含めていません。Core workflow が持つべき操作の形は次の通りです。

python3 -m ol_startlist inspect core-sample/input/joy-entrylist-anonymous.tsv
python3 -m ol_startlist summarize core-sample/input/joy-entrylist-anonymous.tsv --config core-sample/config/sample-2days.yaml
python3 -m ol_startlist validate core-sample/input/joy-entrylist-anonymous.tsv --config core-sample/config/sample-2days.yaml
python3 -m ol_startlist build core-sample/input/joy-entrylist-anonymous.tsv --config core-sample/config/sample-2days.yaml --output-dir core-sample/output
このページのデータは公開説明用の匿名サンプルです。実大会の参加者情報やカード番号は含みません。
